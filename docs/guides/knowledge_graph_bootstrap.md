
## The Concept — Two Types of Knowledge

Your database needs two distinct types of knowledge working together:

```
TYPE 1: HISTORICAL BASELINE (seeded once, static)
"What the world was before your system started watching"
→ History, geography, entities, relationships, declassified intelligence
→ Gives your system CONTEXT for everything it collects live

TYPE 2: LIVE INTELLIGENCE (collected continuously)
"What the world is doing right now"
→ Flight tracks, OSINT articles, seismic events, signals
→ Gains MEANING because the historical baseline exists beneath it
```

Without the baseline, your system is a newborn — it knows nothing about history, geography, or existing entities when it starts. Every entity it discovers is brand new. Every pattern has no historical comparison.

With the baseline, your system wakes up on day one already knowing the world — every country, every major organization, every historical event, every geopolitical relationship — and live collection immediately has context to plug into.

### CATEGORY 1 — THE GEOGRAPHIC WORLD FOUNDATION

Before any intelligence, you need to know where everything is.

**GeoNames — 11 Million Place Names With Coordinates**[^1][^2]

```bash
# Download the entire world's geographic database
# Every city, region, country, mountain, river, sea — with lat/lon
wget https://download.geonames.org/export/dump/allCountries.zip

# What you get:
# - 11 million place names
# - Each with: lat, lon, country, admin region, population, elevation
# - Countries, cities, mountains, rivers, lakes, airports, ports
# - All with ISO codes and PostGIS-ready coordinates

# Ingest into PostGIS as your geographic reference layer
# When an OSINT article mentions "Strait of Malacca" —
# your system already knows exactly where that is
```

**OSMNames — 21 Million Place Names With Bounding Boxes**[^3][^4]

```bash
# More detailed than GeoNames — includes bounding boxes
# Critical for PostGIS: you can query "within this region"
wget https://github.com/geometalab/OSMNames/releases/download/v2.0/planet-latest_geonames.tsv.gz

# ~21 million geographic features
# Each with: full hierarchy (country → region → city)
# Plus: bounding box (vital for spatial queries on globe)
```

**Why this matters for your system:** When your OSINT agent collects an article mentioning "Xinjiang Province," your Entity Agent doesn't have to guess where that is. The geographic baseline gives it an immediate geo-coordinate with a bounding box — PostGIS precision from day one.

This is the most valuable seed source for geopolitical understanding. The CIA has declassified **over 13 million pages** of historical intelligence documents, all publicly available.[^5][^6][^7]

```

URL: https://www.cia.gov/readingroom/

WHAT'S IN THERE:
├── Central Intelligence Bulletins (1947–1990s)
│   Daily intelligence products — same format as your Morning Brief
│   Reading these teaches your system what intelligence looks like
│
├── National Intelligence Estimates (NIEs)
│   Finished all-source assessments on every major geopolitical topic
│   Korea, Vietnam, Soviet Union, Middle East, China
│   These are the gold standard of finished intelligence writing
│
├── Director of Central Intelligence records (1947–1960s)
│   Memos, meeting minutes, operational assessments
│
├── Cold War collections
│   Berlin Tunnel, Cuban Missile Crisis, Korean War,
│   Vietnam-era assessments, Soviet military assessments
│
├── Henry Kissinger papers (40,000 pages)
│
├── Science & Technology R&D (20,000 pages)
│
└── STARGATE program (declassified)
```

**How to ingest this into your system:**

```python
# Each declassified CIA document becomes a HISTORICAL UIR
# source_type = 'OSINT'
# source_agent = 'historical_seed_importer'
# source_url = CIA FOIA document URL
# domain = classified by Kimi K2.5 during ingestion
# geo = extracted from document text by NLP
# embedding = generated by your embedding model
# tags = ['historical', 'declassified', 'CIA', 'CREST']

# When your live OSINT agent collects a news article about
# North Korea's missile program in 2026, the semantic search
# finds CIA assessments of North Korea from 1985 and 1993
# as related UIRs. Historical context instantly available.
```

### CATEGORY 3 — WIKIDATA KNOWLEDGE GRAPH (5 Million Entities)

Wikidata is the world's largest structured knowledge graph — 112 million entities with relationships, all freely downloadable.[^8][^9]

```bash
# Download Wikidata5m — the curated intelligence subset
# 5 million entities × their Wikipedia descriptions × relationships
# Format: entity_id, relation, entity_id (triple format)

# What you get:
# Every country and its government entities
# Every major organization (corporations, NGOs, military)
# Every significant person in history
# Every major event
# All their relationships to each other

# Example triples already in Wikidata:
# (Al-Qaeda) → FOUNDED_BY → (Osama bin Laden)
# (Strait of Hormuz) → LOCATED_IN → (Persian Gulf)
# (Boeing) → MANUFACTURES → (747)
# (Singapore) → MEMBER_OF → (ASEAN)
```

**Import strategy — seed your entity graph before day one:**

```python
# Wikidata import = Phase 2 entity graph pre-populated
# Before your system collects a single live record,
# it already knows about 5 million entities and their relationships

# For each Wikidata entity of interest:
entity = {
    "name": "Strait of Hormuz",
    "entity_type": "LOCATION",
    "canonical_name": "Strait of Hormuz",
    "description": "Strategic waterway between Persian Gulf and Gulf of Oman",
    "primary_geo": POINT(56.5, 26.6),  # PostGIS coordinate
    "confidence": 0.99,
    "watch_status": "PASSIVE",
    "source": "wikidata_seed",
    "uir_refs": []  # empty until live collection adds to it
}

# Now when your GEOINT agent detects a vessel anomaly
# near these coordinates, it immediately links to this
# pre-existing entity — no discovery needed
```

## THE INGESTION ARCHITECTURE

How do you actually load all of this into your system? Kimi K2.5 does the processing. The pipeline looks like this:

```python
# SEED INGESTION PIPELINE
# Run once before live collection begins
# Cortex manages this as a long-running startup task

class SeedIngestor:

    def ingest_geonames(self):
        # 11M rows → entities table (LOCATION type)
        # Each row = one entity with PostGIS coordinate
        # Fast bulk insert via COPY command
        # ~2 hours to complete

    def ingest_wikidata(self):
        # 5M entities → entities table
        # 50M relationships → entity_relationships table
        # Filter: only entities with >10 Wikipedia mentions
        # (reduces noise, keeps significant entities)
        # ~6 hours to complete

    def ingest_cia_crest(self, document):
        # Each CIA document → intelligence_records UIR
        # Kimi K2.5 extracts:
        #   - geo location (what region/country)
        #   - entities mentioned (people, orgs, places)
        #   - domain (MILITARY, POLITICAL, etc.)
        #   - time period (when the document refers to)
        #   - 2-sentence summary (content_summary)
        #   - headline (content_headline)
        # Then generates embedding
        # ~775,000 documents × 30 seconds each
        # = weeks of processing time
        # But: runs in background, you don't wait

    def ingest_gutenberg(self, book, category):
        # Each book → chunked into 2,000-token passages
        # Each passage → one UIR with:
        #   source_type = 'OSINT'
        #   tags = ['historical', 'literature', category]
        #   content_raw = passage text
        #   embedding = generated
        # Enables historical RAG retrieval

    def ingest_acled(self):
        # Every armed conflict event → UIR
        # Already has: date, location, coordinates, actors
        # Perfect UIR seed — structured, geo-tagged, historical

    def ingest_world_bank(self, country, indicator):
        # Each time series → UIRs with:
        #   domain = 'FINANCIAL' or 'POLITICAL'
        #   geo = country centroid
        #   metadata = {year, value, indicator}
```

## What This Means for Day One of Live Collection

When your GEOINT agent detects an unusual flight pattern over the Persian Gulf on day one of live collection, your system doesn't encounter the Persian Gulf as an unknown location. It already knows:

- The Strait of Hormuz is a critical chokepoint (Wikidata + geographic baseline)
- 20% of global oil transits through it (World Bank economic data)
- Historical CIA assessments from 1980–1990 document Iran's strategic interest in controlling it
- ACLED records 340+ conflict events in the region since 1997
- Three entities in your Wikidata graph are flagged as operating in the corridor

The live flight anomaly lands into a world that already has **decades of context** surrounding it. The Analyst Agent's first correlation on day one is richer than most systems achieve after a year of collection.

That's what seeding does. You don't start from zero. **You start from history.**
